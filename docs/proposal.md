# 워해머 40K 전투 확률 계산기 프로젝트 프로포절

## 프로젝트 목적

워해머 40,000 미니어처 게임에서는 부대 편성(List Building), 전술 수립,
교전 결과 예측을 위해 다양한 유닛 간의 전투 결과를 계산해야 한다. 그러나
현재는 공식 룰북, 데이터시트, 공식 앱 또는 Wahapedia, 40k.app 등의
자료를 참고하여 플레이어가 직접 스탯을 확인하고 계산해야 하는 과정이
반복된다.

기존 계산기 또한 공격 및 방어 유닛의 스탯을 직접 입력하는 방식이
대부분이어서 데이터시트를 함께 확인해야 하는 불편함이 존재한다.

본 프로젝트는 **공격 유닛과 방어 유닛을 선택하는 것만으로 실제 게임
규칙을 반영한 전투 결과의 기대값과 확률분포를 계산할 수 있는 도구를 개발하여,
플레이어가 전략 수립에 필요한 정보를 보다 쉽고 빠르게 얻을 수 있도록
하는 것을 목적**으로 한다.

------------------------------------------------------------------------

## 프로젝트 목표

-   최신 데이터시트 기반의 유닛 및 무기 정보를 관리할 수 있는 데이터
    구조를 구축한다.
-   공격 유닛과 방어 유닛을 선택하는 방식으로 전투 계산을 수행할 수
    있도록 구현한다.
-   명중(Hit), 상처(Wound), 내성 굴림(Saving Throw), 피해(Damage) 등
    실제 게임의 전투 절차를 반영한 계산 엔진을 구현한다.
-   특수 규칙과 능력(Abilities), 버프·디버프를 적용할 수 있도록
    설계한다.
-   평균 유효 피해량, 평균 파괴 모델 수, 결과별 피해 확률, 전멸 확률 등
    전투 결과의 기대값과 확률 분포를 직관적으로 제공한다.
-   데이터 업데이트 시 최소한의 수정으로 최신 규칙을 반영할 수 있는
    구조를 설계한다.

## 계산 결과 산출 원칙

본 프로젝트는 한 번의 무작위 전투 결과를 보여 주는 방식보다, 동일한 조건에서
발생할 수 있는 모든 결과의 확률을 계산하여 사용자가 전술적 의사결정에 활용할
수 있는 통계 정보를 제공하는 것을 우선한다.

- 핵심 요약에는 평균 유효 피해량, 평균 파괴 모델 수, 가장 가능성이 높은 결과,
  적 유닛 전멸 확률을 표시한다.
- 세부 결과에는 `정확히 N개 모델을 파괴할 확률`과 `N개 이상 모델을 파괴할
  누적 확률`을 구분하여 제공한다.
- 다중 운드 모델을 공격하는 경우에는 모델 파괴 수뿐 아니라 다음 생존 모델에
  남은 피해까지 포함한 상태별 확률을 계산한다.
- 주사위 결과와 피해 할당 결과는 정규분포를 따른다고 보장할 수 없으므로,
  정규분포를 가정한 그래프 대신 실제 이산 확률분포를 막대그래프나 표로
  시각화한다. 필요할 경우 기대값, 표준편차, 중앙값, 백분위 범위를 보조 지표로
  제공한다.
- `3개 모델 파괴 + 2 운드 피해`와 같은 정수 상태는 평균값이 아니라 최빈값이나
  중앙값에 해당할 수 있으므로, 화면에서는 `가장 가능성이 높은 결과` 또는
  `대표 결과`로 명확히 구분한다.

예시 결과는 다음과 같이 구성한다.

- 평균 유효 피해: 17.6W
- 평균 파괴 모델 수: 3.2개
- 가장 가능성이 높은 결과: 모델 3개 파괴 + 다음 모델에 2W 피해
- 정확히 모델 1개 파괴: n%
- 정확히 모델 2개 파괴: m%
- 모델 3개 이상 파괴: x%
- 적 유닛 전멸: a%

## 트러블 슈팅

### 1. 복잡한 게임 규칙의 모델링

워해머 40k는 명중, 상처, 내성 판정 외에도 치명타, 재굴림, 추가 명중,
필 노 페인(Feel No Pain), 피해량 감소 등 다양한 특수 규칙과 예외 규칙이
존재한다. 이러한 규칙들은 서로 중첩되거나 적용 순서에 따라 결과가 달라질
수 있으므로, 단순한 수식만으로 모든 상황을 처리하기 어렵다.

#### 해결 방식

- 전투 절차를 `공격 횟수 결정 → 명중 → 상처 → 내성 → 피해 적용 → 사상자
  처리`와 같은 단계별 파이프라인으로 분리한다.
- 각 특수 규칙은 계산식 내부에 직접 작성하지 않고, 특정 단계에 개입하는
  독립적인 효과 객체로 정의한다. 예를 들어 명중 재굴림은 명중 단계,
  관통력 변경은 내성 단계에만 적용되도록 구성한다.
- 효과마다 적용 조건, 적용 시점, 우선순위, 중첩 가능 여부를 명시하여 규칙
  간 충돌을 일관된 기준으로 처리한다.
- 계산 과정에서 사용되는 전투 상태는 가능한 한 불변 데이터로 관리하고,
  각 단계가 새로운 상태를 반환하도록 설계하여 규칙 적용 과정을 추적할 수
  있도록 한다.
- 대표적인 공식 예시와 실제 플레이 상황을 테스트 케이스로 작성하고,
  규칙이 추가되거나 수정될 때 기존 계산 결과가 변하지 않는지 회귀 테스트를
  수행한다.

**적용 기술·설계 인사이트:** 규칙 엔진(Rules Engine), 파이프라인 패턴,
전략 패턴(Strategy Pattern), 책임 연쇄 패턴(Chain of Responsibility), 불변
상태 관리, 단위 테스트 및 골든 테스트(Golden Test)

### 2. 지속적인 데이터 업데이트 대응

게임 밸런스 패치와 신규 데이터시트 출시로 유닛의 능력치, 무기 프로필,
포인트, 특수 규칙이 지속적으로 변경된다. 계산 로직에 이러한 값을 직접
작성할 경우 업데이트마다 프로그램을 수정하고 다시 배포해야 하는 문제가
발생한다.

#### 해결 방식

- 계산 엔진과 게임 데이터를 분리하고, 유닛·무기·능력 데이터를 JSON,
  YAML 또는 데이터베이스와 같은 외부 데이터 소스로 관리한다.
- 유닛, 무기, 능력, 진영, 데이터 버전 사이의 관계를 명확히 정의한 공통
  데이터 스키마를 설계한다.
- 각 데이터에 출처, 규칙 버전, 적용 시작일, 마지막 수정일을 기록하여 어떤
  기준의 데이터로 계산했는지 확인할 수 있도록 한다.
- 신규 데이터를 불러올 때 필수 필드 누락, 잘못된 수치 범위, 존재하지 않는
  능력 참조 등을 자동으로 검사하는 검증 절차를 구축한다.
- 데이터 변경 전후의 계산 결과를 비교하는 자동 테스트를 구성하고, 문제가
  발생하면 이전 버전으로 되돌릴 수 있도록 데이터 버전과 변경 이력을
  보존한다.
- 초기 개발 단계에서는 수동 검수된 데이터 파일을 사용하고, 이후 필요에
  따라 관리자용 편집 화면이나 데이터 변환 파이프라인을 추가한다.

**적용 기술·설계 인사이트:** 데이터 주도 설계(Data-driven Design), JSON
Schema 또는 런타임 스키마 검증, 데이터베이스 마이그레이션, 버전 관리,
ETL 파이프라인, 변경 이력 및 롤백, CI 기반 데이터 검증

### 3. 다양한 전투 상황의 처리

무기 옵션, 장비 변경, 특수 능력, 버프와 디버프, 대상 조건 등 전투 결과에
영향을 주는 요소가 많다. 모든 조합을 별도의 계산식으로 구현하면 조건문이
지나치게 증가하고, 새로운 규칙을 추가할 때 기존 로직에 부작용이 발생할 수
있다.

#### 해결 방식

- 공격자, 방어자, 선택 무기, 거리, 적용 중인 효과 등을 하나의
  `BattleContext`로 구성하여 모든 계산 단계가 동일한 전투 정보를 참조하게
  한다.
- 버프와 디버프는 능력치를 직접 덮어쓰지 않고 `+1 명중`, `상처 굴림 재굴림`,
  `피해량 최소값 변경`과 같은 선언형 Modifier로 변환하여 적용한다.
- Modifier마다 대상, 적용 조건, 지속 범위, 중첩 규칙, 상한·하한을 정의하여
  여러 효과가 동시에 적용되어도 동일한 규칙으로 계산되도록 한다.
- 각 계산 단계의 가능한 결과와 발생 확률을 이산 확률분포로 표현하고, 명중,
  상처, 내성, 피해 적용 단계의 분포를 순차적으로 합성한다.
- 공격 횟수나 피해량이 주사위로 결정되는 경우에도 단일 난수를 생성하지 않고,
  가능한 값과 확률을 모두 다음 계산 단계로 전달한다.
- 모델별 피해 할당과 초과 피해 소멸을 반영하기 위해 계산 상태를
  `파괴된 모델 수`, `현재 모델의 남은 운드`, `남은 공격 수` 등으로 정의하고,
  상태 전이별 확률을 누적한다.
- 동일한 상태에 도달한 계산 경로는 동적 계획법과 메모이제이션을 통해 하나로
  병합하여 조합 폭증을 완화한다. 확률이 매우 낮은 상태를 정리할 경우에는
  허용 오차 기준을 명시한다.
- 계산 결과는 단일 랜덤 표본이 아니라 기대값, 최빈 결과, 결과별 확률, 누적
  확률로 반환하며, 동일한 입력에는 항상 동일한 결과가 나오도록 한다.
- 복잡한 규칙의 구현 정확성은 공식 예시, 수작업 계산, 제한적인 내부 반복
  시뮬레이션과 비교하여 검증할 수 있지만, 사용자에게는 무작위 표본이 아닌
  계산된 확률분포를 제공한다.

**적용 기술·설계 인사이트:** 전투 컨텍스트 객체, 선언형 Modifier 시스템,
태그 기반 조건 판정, 이산 확률분포, 확률 질량 함수(PMF), 확률분포 합성,
동적 계획법, 상태 전이 모델, 메모이제이션 및 캐싱, 속성 기반 테스트
(Property-based Testing)

### 4. 사용 편의성 확보

기존 계산기들은 대부분 사용자가 공격자와 방어자의 스탯을 직접 입력해야
하므로 데이터시트를 함께 확인해야 하는 번거로움이 있다. 반대로 모든 세부
설정을 한 화면에 제공하면 초보 사용자에게 복잡한 인터페이스가 될 수 있다.

#### 해결 방식

- 기본 흐름을 `공격 유닛 선택 → 무기 선택 → 방어 유닛 선택 → 결과 확인`의
  최소 단계로 구성한다.
- 진영, 키워드, 유닛명 검색과 자동 완성을 제공하여 원하는 유닛을 빠르게
  찾을 수 있도록 한다.
- 유닛과 무기를 선택하면 일반적으로 사용되는 모델 수, 무기 수, 사거리,
  기본 장비를 자동으로 설정하고 사용자가 필요한 부분만 수정하도록 한다.
- 버프, 디버프, 지형, 특수 규칙과 같은 고급 조건은 별도 영역에 배치하는
  점진적 공개(Progressive Disclosure) 방식을 적용한다.
- 결과 화면의 요약 영역에는 평균 유효 피해량, 평균 파괴 모델 수, 가장
  가능성이 높은 피해 상태, 전멸 확률을 우선 표시한다.
- 세부 영역에는 명중 성공 수, 상처 성공 수, 실패한 내성 수, 최종 피해량의
  기대값을 단계별로 제공하여 계산 근거를 확인할 수 있도록 한다.
- 모델 파괴 확률은 `정확히 N개 파괴`와 `N개 이상 파괴`를 구분하고, 전체
  결과를 이산 확률분포 막대그래프와 표로 시각화한다.
- 결과의 변동성을 설명할 필요가 있는 경우 표준편차와 백분위 범위를 함께
  제공하되, 정규분포를 가정하지 않는다.
- 사용한 조건을 프리셋 또는 공유 가능한 링크로 저장하여 동일한 전투 상황을
  반복해서 비교할 수 있도록 한다.
- 모바일 환경에서도 핵심 기능을 사용할 수 있도록 반응형 UI와 충분한 터치
  영역을 제공한다.

**적용 기술·설계 인사이트:** 검색 자동 완성, 스마트 기본값, 점진적 공개,
폼 상태 관리, 반응형 웹 디자인, 결과 설명 가능성(Explainability), 데이터
시각화, URL 상태 직렬화 및 프리셋 저장

## 구현 및 운영 계획

본 프로젝트는 별도의 백엔드 서버와 외부 데이터베이스를 운영하지 않는 정적
웹 애플리케이션으로 구현한다. 사용자는 웹사이트에서 공격 유닛, 무장, 방어
유닛과 전투 조건을 선택하며, 전투 계산은 사용자의 브라우저에서 수행한다.

초기 기술 스택은 React, TypeScript, Vite를 사용한다. 사용자 인터페이스와
전투 계산 엔진은 서로 분리하고, 계산 엔진은 입력된 전투 조건과 게임 데이터를
받아 확률분포와 요약 결과를 반환하는 독립 모듈로 구성한다. 이를 통해 화면
구성이나 데이터 저장 방식이 변경되더라도 핵심 계산 로직을 재사용할 수 있도록
한다.

### 게임 데이터 관리

유닛, 모델, 무장, 능력 및 규칙 데이터는 계산 프로그램과 분리하여 관리한다.
각 데이터에는 변경되지 않는 고유 ID와 적용 버전을 부여하고, 게임 버전과
진영을 기준으로 파일을 분할한다.

최초 접속 시 모든 게임 데이터를 한 번에 내려받지 않고, 사용자가 선택한
진영과 계산에 필요한 데이터만 내려받는다. 내려받은 데이터는 브라우저의 로컬
저장소에 보관하여 이후 접속에서는 기존 데이터를 우선 사용한다.

게임 업데이트가 발생하면 웹사이트는 소형 버전 정보 파일을 확인하고, 로컬에
저장된 데이터와 비교하여 변경된 진영의 데이터만 다시 내려받는다. 이전 버전의
데이터도 별도로 보존하여 사용자가 데이터 버전을 전환하며 과거 게임 환경의
전투 결과를 계산할 수 있도록 한다.

### 데이터 갱신 도구

게임 업데이트를 서비스에 반영하기 위해 관리자용 데이터 갱신 도구를 별도로
구현한다. 초기에는 별도의 관리자 웹사이트 대신 개발 환경에서 실행하는 로컬
도구를 사용한다.

관리자는 CSV 또는 구조화된 원본 데이터에서 변경된 유닛과 무장 정보를
수정한다. 데이터 갱신 도구는 입력값과 데이터 간 참조 관계를 검사하고, 이전
버전과의 변경점을 비교한 뒤 웹사이트에서 사용할 데이터 파일과 버전 정보를
자동으로 생성한다.

이를 통해 게임 데이터가 변경될 때 계산 프로그램의 코드를 직접 수정하는
작업을 최소화하고, 잘못된 데이터가 배포되는 문제를 방지한다. 레코드 단위의
증분 패치 기능과 관리자용 편집 화면은 실제 데이터 크기와 업데이트 빈도를
확인한 이후 확장 기능으로 검토한다.

### 배포 및 비용

소스 코드와 게임 데이터는 Git과 GitHub를 통해 버전 관리하고, 무료 정적 웹
호스팅 서비스를 이용하여 배포한다. 초기 배포 환경은 Cloudflare Pages와 같은
무료 서비스를 사용하며, 서비스에서 기본 제공하는 주소를 사용하여 별도의
도메인 비용이 발생하지 않도록 한다.

프로그램이나 데이터가 변경되면 원격 저장소에 변경 사항을 반영하고, 자동
테스트와 빌드를 거쳐 웹사이트를 다시 배포한다. 사용자의 브라우저에서는
변경되지 않은 파일과 데이터는 기존 캐시를 사용하고, 실제로 변경된 파일만
새로 내려받도록 구성한다.

초기 구현 범위에서는 다음과 같은 유료 또는 서버 기반 요소를 제외한다.

- 백엔드 서버 및 외부 데이터베이스
- 사용자 계정과 클라우드 동기화
- 유료 API와 유료 파일 저장소
- 개인 도메인과 유료 분석 서비스
- 실시간 다중 사용자 기능

이를 통해 개발 및 운영 비용을 발생시키지 않는 것을 기본 목표로 한다.

## 단계별 개발 계획

웹 개발과 배포 경험이 적은 상황에서 전체 기능을 한 번에 구현하면 문제의
원인을 구분하기 어렵다. 따라서 핵심 계산 로직을 먼저 검증한 뒤 데이터와
배포 기능을 순차적으로 확장한다.

### 1단계: 계산 엔진 프로토타입

- 일부 유닛과 무기 데이터를 코드 내부에 임시로 정의한다.
- 명중, 상처, 내성, 고정 피해와 모델별 피해 할당을 구현한다.
- 평균 피해량과 기본 이산 확률분포를 산출한다.
- 수작업 계산 및 제한적인 반복 시뮬레이션과 결과를 비교한다.

### 2단계: 기본 웹 인터페이스

- 공격 유닛, 무기, 방어 유닛을 선택하는 기본 화면을 구현한다.
- 전투 조건을 입력하고 결과 요약과 확률분포를 표시한다.
- 모바일 환경을 고려한 기본 반응형 레이아웃을 적용한다.

### 3단계: 게임 데이터 분리

- 유닛, 무기, 능력 데이터를 외부 데이터 파일로 분리한다.
- 공통 데이터 스키마과 고유 ID 체계를 정의한다.
- 입력 데이터의 형식과 참조 관계를 자동으로 검증한다.

### 4단계: 로컬 저장 및 버전 관리

- 필요한 진영 데이터를 선택적으로 내려받는다.
- 내려받은 데이터를 브라우저에 저장하고 재사용한다.
- 게임 데이터 버전 선택과 변경 데이터 갱신 기능을 구현한다.

### 5단계: 데이터 갱신 및 자동 배포

- 원본 데이터를 검증하고 배포용 파일로 변환하는 도구를 구현한다.
- 이전 버전과의 변경점 및 오류를 확인할 수 있도록 한다.
- GitHub와 정적 호스팅 서비스를 연결하여 테스트, 빌드, 배포를 자동화한다.

### 6단계: 규칙 및 사용성 확장

- 재굴림, 치명타, 추가 명중, 피해 감소, Feel No Pain 등 복합 규칙을
  단계적으로 추가한다.
- 버프·디버프, 검색, 자동 완성, 프리셋 및 공유 기능을 확장한다.
- 한국어와 영어를 시작으로 사용자 인터페이스와 게임 데이터 표시명을 분리한
  다국어 지원 기능을 추가한다.
- 실제 사용자 피드백과 성능 측정 결과에 따라 데이터 패치 방식과 관리 도구를
  개선한다.

## 초기 구현 범위

초기 MVP에서는 프로젝트의 핵심 가치인 전투 계산의 정확성과 데이터 관리
가능성을 검증하는 데 집중한다.

### 포함 범위

- 정적 웹 애플리케이션
- 공격 및 방어 유닛 선택
- 무장과 기본 전투 조건 설정
- 명중, 상처, 내성, 피해 및 모델 파괴 계산
- 평균 피해량과 실제 이산 확률분포 제공
- 진영별 게임 데이터 분리
- 브라우저 로컬 저장과 데이터 버전 선택
- 데이터 검증 및 갱신 도구
- 무료 정적 호스팅을 통한 배포

### 초기 제외 범위

- 모든 유닛과 특수 규칙의 완전한 지원
- 사용자 계정과 서버 저장
- 관리자용 웹 편집 화면
- 다국어 사용자 인터페이스 및 번역 데이터 관리
- 실시간 협업 및 다중 사용자 기능
- 레코드 단위 증분 패치
- 유료 서비스와 개인 도메인

초기 제외 항목은 실제 사용 과정에서 필요성과 우선순위를 확인한 뒤 후속
버전에서 검토한다.

## 기대 효과

본 프로젝트를 통해 플레이어는 데이터시트를 반복해서 확인하거나 복잡한
확률 계산을 직접 수행하지 않고도 특정 유닛과 무장의 전투 성능을 비교할 수
있다. 단일 무작위 결과가 아니라 평균값과 결과별 확률을 제공하므로 부대 편성,
공격 대상 선택, 자원 투입 여부와 같은 전술적 판단에 활용할 수 있다.

개발 측면에서는 확률 계산, 상태 전이, 규칙 엔진, 데이터 주도 설계, 테스트
자동화, 브라우저 저장소, 정적 웹 배포와 같은 기술을 하나의 프로젝트에서
경험할 수 있다. 또한 계산 엔진, 게임 데이터, 사용자 인터페이스와 데이터
갱신 도구를 분리함으로써 장기적인 유지보수와 기능 확장이 가능한 구조를
확보한다.

## 프로젝트 완료 기준

초기 버전은 다음 조건을 충족할 때 완료된 것으로 판단한다.

- 지원 대상 유닛과 무장을 화면에서 선택할 수 있다.
- 기본 전투 절차와 우선 지원 대상으로 선정한 특수 규칙이 정확하게 계산된다.
- 평균 피해량, 평균 파괴 모델 수, 결과별 확률과 전멸 확률을 확인할 수 있다.
- 동일한 입력과 동일한 데이터 버전에는 항상 동일한 계산 결과가 반환된다.
- 게임 데이터가 계산 로직과 분리되어 있으며, 검증된 갱신 절차를 통해 새로운
  버전을 추가할 수 있다.
- 필요한 데이터가 브라우저에 저장되고, 변경된 데이터만 다시 내려받을 수
  있다.
- 이전 게임 데이터 버전을 선택하여 계산할 수 있다.
- 자동 테스트를 통과한 정적 웹사이트를 무료 호스팅 환경에 배포할 수 있다.

상세한 데이터 구조, 계산 엔진 내부 설계, 로컬 저장 방식, 캐시 정책 및 배포
자동화 절차는 별도의 기술 설계서에서 정의한다.
