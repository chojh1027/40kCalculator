## 2. 현재 단계 평가

현재 프로젝트는 **가변 공격과 가변 피해를 포함하는 기본 전투 확률 계산 MVP, 선택형 웹 UI와 대표 회귀·불변식 테스트가 구현된 단계**다.

계산 엔진은 고정값 또는 가변 주사위 공격 횟수와 피해를 사용하는 단일 공격 그룹에 대해 명중, 상처, 내성, 피해 할당을 실제 이산 확률분포로 계산한다. 각 실패한 내성은 독립적인 피해 이벤트이며, 다중 운드 모델과 일반 피해의 초과 피해 비전달(non-spill)을 처리한다.

`BattleInput.attacks`와 `BattleInput.damage`는 기존 숫자 입력과 `DiceExpression`을 모두 지원한다. 공격 횟수별 명중·상처·내성 분포와 실패한 내성별 피해 상태 전이를 PMF로 합성하며, 같은 방어 상태에 도달하는 경로를 병합한다.

여러 모델이 같은 가변 공격 무장을 사용하면 `repeatAttackCount`로 각 모델의 독립 공격 주사위를 합산한다. 피해 주사위는 모델 수로 반복하지 않고, 실제 실패한 내성마다 새로 적용한다.

검증 계층은 다음과 같이 구성됐다.

- 규칙 단위·회귀 테스트: `index.test.ts`
- 대표 전투 골든 테스트: `golden.test.ts`
- 확률·상태·결정성·단조성 불변식 테스트: `invariants.test.ts`

현재 임시 검증 데이터:

- `D6 Test Blaster (Temporary)`: 모델당 D6 공격
- `D6 Damage Cannon (Temporary)`: 공격 1, 피해 D6

두 무장은 계산 경로 검증용이며 정식 게임 데이터가 아니다.

| GitHub Actions·Pages | ✅ | 검사와 정적 배포 구성 |

### 현재 핵심 제약

- 단일 공격 그룹만 지원한다.
- 재굴림, 치명타, 추가 명중, 치명상, 피해 감소, Feel No Pain을 지원하지 않는다.
- 임시 D6 무장은 정식 게임 데이터가 아니다.
- 유닛과 모델 프로필이 하나의 `Unit` 타입에 결합돼 있다.
- 외부 JSON, 런타임 데이터 검증, 데이터 버전과 IndexedDB가 없다.
- 의존성 lockfile 정책이 확정되지 않아 설치 재현성이 충분히 고정되지 않았다.
- 속성 기반 테스트 프레임워크는 아직 도입하지 않았다.

### 단계 5: 골든·불변식 테스트 강화

상태: ✅ 완료

구현 파일:

```text
packages/calculator/src/golden.test.ts
packages/calculator/src/invariants.test.ts
```

골든 케이스:

- 고정 공격 + 고정 피해
- D6 공격 + 고정 피해
- 고정 공격 + D3 피해
- 고정 공격 + D6 피해
- D6 공격 + D6 피해

검증 불변식:

- 모든 공개 분포의 확률 합은 1
- 확률은 유한한 0~1 범위
- 동일 값과 동일 방어 상태는 중복 출력되지 않음
- 방어 상태의 파괴 모델 수와 잔여 운드는 유효 범위
- 요약 기대값과 분포 기대값 일치
- 동일 입력의 결과 결정성
- 방어 내성이 개선되면 평균 피해와 전멸 확률이 증가하지 않음
- 방어 모델 수가 증가하면 전멸 확률이 증가하지 않음
- 일반 피해 이벤트 하나는 모델 두 개 이상을 파괴하지 않음
- 공격 0회는 모든 피해 표현에서 최종 피해 0
